## 內容大綱

隨著 AI 生態繁榮發展，每家公司、每個產品都在開發自己的 AI 工具與外掛系統。這導致開發者如果要讓自己的服務能被 ChatGPT、Claude、Dify 等不同的 AI 平台使用，就必須分別為每一個平台寫一次串接代碼。

為了解決這種碎片化與重複勞動的問題，Anthropic 提出了 **MCP (Model Context Protocol)** 協議。

### 1. 什麼是 MCP？

MCP 是一個開源的標準協議，旨在規範大語言模型 (LLM) 應用戶端 (Client) 與外部資料來源或工具伺服器 (Server) 之間的資料交換格式與連線方式。

您可以把它想像成是 AI 界的 **USB-C 介面**。
過去每種手機都要配對一條專屬形狀的充電線。現在只要大家都遵守同一種接頭標準 (MCP)，不管是哪一家的手機 (AI 應用)，插上同一條線就能直接讀取資料或呼叫工具 (MCP Server)。

### 2. MCP 解決了什麼問題？

1.  **一次開發，到處可用**：開發者只要把自己的系統封裝成一個標準的 MCP Server，未來任何支援 MCP 的 AI 客戶端（如 Dify、Claude Desktop、Cursor 等）都可以直接連線並使用您提供的工具與資料，無需任何修改。
2.  **增強安全性**：MCP 支援本地雙向溝通。您可以讓 AI 應用跑在雲端，而 MCP Server 跑在企業內網或本地電腦，實現安全的隔離，保護敏感資料庫不對外網曝露。
3.  **清晰的責任劃分**：AI 應用戶端只管理解意圖、維持對話上下文；而資料檢索與工具執行的重責大任則全交由專業的 MCP Server 處理。

### 3. MCP 的基本架構

MCP 採用 Client-Server 架構：

*   **MCP 客戶端 (Client)**：負責發起對話、持有大型語言模型 (LLM)。例如 Dify 就是一個強大的 MCP Client。
*   **MCP 伺服器 (Server)**：負責提供特定領域的資料或工具執行能力。它可以是存取本地檔案的服務、連線 Github 的服務、或是操作 Slack 傳訊息的服務。

MCP 伺服器通常會提供三種主要能力 (Capabilities) 給 客戶端：
1.  **資源 (Resources)**：提供靜態或動態的資料內容供讀取 (例如：文件、資料庫內容)。
2.  **提示模板 (Prompts)**：提供預置的系統提示詞範本。
3.  **工具 (Tools)**：讓 LLM 能夠主動呼叫並執行的函數 (例如：執行一段代碼、發送一封電子郵件)。
