完成本單元後，您將能夠：

- 明白如何取得 MCP 端點 (Endpoint) 與金鑰供外部呼叫

## 內容大綱

在前面的單元中，我們學會了如何使用寫程式的方式來開發 MCP Server。然而，**Dify 最強大的地方在於，它已經內建了將任何工作流 (Workflow) 變成 MCP 服務的能力！**
這意味著，您完全不寫一行程式碼，只要在畫面上拖拉拽組合好的 AI 工具與邏輯，就能直接變成一個標準的 MCP 伺服器，造福整個組織或世界。

### 1. 為什麼要將流程轉為 MCP 服務？

假設您在 Dify 中花了很多心血做了一個「分析應徵者履歷並自動寄送面試邀請信」的超強工作流。
原本您只能在 Dify 的網頁介面裡點擊使用。現在透過開啟 MCP 功能，您可以把這個超強大腦變成一個「工具」。其他同事只要使用支援 MCP 的應用程式（像 Cursor 或 Claude Desktop），就可以直接喚起您的這個大作來幫他們看履歷。

### 2. 在 Dify 操作步驟：一鍵開啟 MCP Server

Dify 把這個轉換過程做得無比簡單：

1.  **進入您的工作流 (Workflow)**：打開您已經設計好、並確保輸入 (Start) 與輸出 (End) 節點設定完整的工作流。
2.  **開啟發佈設定**：點擊工作流編輯介面右上方的「發佈 (Publish)」下拉選單。
3.  **啟用 MCP 服務**：在發佈選項中，找到並開啟「啟用 MCP 服務 (Enable MCP Server)」的開關。
4.  **填寫 MCP 工具描述**：系統會要求您為這個 MCP 工具填寫名稱與詳細描述。
    *(💡 溫馨提示：請詳盡填寫描述！因為外部的 AI 將完全憑藉這段文字來決定什麼時候要呼叫這個工具。)*
5.  **取得連線資訊**：設定完成後，Dify 會給您一組專屬的 **API URL** 以及 **API Key (金鑰)**。這就是您心血結晶的 MCP 連線點！

### 3. 如何驗證 MCP 服務狀態？

您可以想像，現在您的 Dify 伺服器就在針對您剛剛發布的工作流，提供了一個 HTTP SSE (Server-Sent Events) 長連線通道。

這就結束了！您不需要自己寫 Python FastMCP，也不需要架設 Server。Dify 已經幫您把這個工作流變成了一個符合全球通用標準協議的伺服器。只要拿到這組 URL 跟 API Key，任何人都可以隨時取用這個驚人的 AI 流程。

在下一個單元，我們將看看有了這組 MCP URL 後，外部應用到底該怎麼整合呼叫。
