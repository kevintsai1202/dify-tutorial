## 內容大綱

在 Dify 的工作流中，當您需要將多個變數組合、格式化或轉換成特定文字結構時，除了使用代碼節點，更輕量且直覺的選擇是 **模板轉換節點 (Template Node)**。

### 1. 什麼是模板轉換節點？

模板轉換節點使用 **Jinja2** 模板引擎的語法。它可以讓您在一段預先寫好的文本模板中，插入變數、甚至是加入簡單的邏輯判斷和迴圈，最終渲染出一段純文字。

**常見應用場景**：
1.  **動態生成 Prompt**：將使用者的問題、檢索到的多筆知識庫背景資料、以及當前時間，組合成一段完整的 Prompt 餵給 LLM 節點。
2.  **生成特定格式資料**：例如將上游傳來的文章列表，轉換成 Markdown 列表格式，或者是符合特定 API 要求的 JSON 字串。
3.  **自動化回覆排版**：將多個處理步驟的結果（如天氣預報、股價資訊、新聞摘要）組合成一篇排版精美的日報，再一次性發送。

### 2. Jinja2 基礎語法簡介

在 Dify 中，您可以直接在模板編輯區輸入純文本，並利用以下標籤來插入動態內容：

1.  **輸出變數**：`{{ variable_name }}`
2.  **條件判斷 (if/else)**：
    ```jinja2
    {% if score >= 60 %}
    及格了！
    {% else %}
    不及格，請重修。
    {% endif %}
    ```
3.  **迴圈 (for)**：
    ```jinja2
    今日新聞摘要：
    {% for news in news_list %}
    - {{ news.title }} ({{ news.date }})
    {% endfor %}
    ```

### 3. 讓 AI 協助產生 Jinja2 語法

如果遇到較為複雜的變數結構（例如巢狀的 JSON 物件），要手寫出正確的 Jinja2 迴圈可能會讓人抓狂。此時可以藉助 AI 大語言模型來幫忙：

**向 AI 提問的 Prompt 範例：**
```text
我正在 Dify 中使用 Jinja2 模板節點。
我有以下 JSON 格式的變數資料（變數名稱為 `api_response`）：
[
  {"name": "Apple", "price": 30, "stock": true},
  {"name": "Banana", "price": 15, "stock": false},
  {"name": "Cherry", "price": 100, "stock": true}
]

請幫我寫一段 Jinja2 代碼，遍歷這個 api_response。
要求：
1. 如果 stock 為 true，印出："產品：[名字]，價格：[價格]，有現貨"
2. 如果 stock 為 false，印出："產品：[名字]，缺貨中"
3. 每一個產品獨立一行。
```

AI 生成的結果可以直接貼入 Dify 的模板節點中，就能快速完成資料的格式化與渲染，是拼接複雜字串的絕佳工具！
